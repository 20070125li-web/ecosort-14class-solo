## ⚠️ 问题诊断：类别不平衡，非数据泄露

### 为什么不是数据泄露？

1. **train/val 划分是正确的**
   - 随机种子固定（seed=42）
   - 分离的数据集

2. **数据增强不会造成泄露**
   - 只在训练时使用
   - 验证时只做 resize 和 normalize

3. **文件名不会造成泄露**
   - 模型看不到文件名
   - 只处理图像像素数据

---

## 🎯 真正的问题：严重的类别不平衡

### 数据分布：
```
recyclable: 2000 张 (93.6%)  ← 多数类
other:       137 张 (6.4%)   ← 少数类
```

### 如果模型全部预测为 recyclable：

| 指标 | 值 | 说明 |
|------|-----|------|
| Accuracy | 2000/2137 = **93.6%** | ✅ 看起来很高！ |
| Recall (other) | 0/137 = **0%** | ❌ 完全漏检！ |
| Precision (other) | 0% | ❌ |
| **F1-Score** | **~0.5** | ❌ 反映真实性能 |

---

## 💡 为什么 F1-Score 这么低？

### F1-Score 计算公式：
```
F1 = 2 × (Precision × Recall) / (Precision + Recall)
```

### 对于 other 类别：
- **Recall** = 0（模型基本没预测 other）
- **Precision** ≈ 0（预测为 other 的样本大多错了）
- **F1 ≈ 0**

### 对于 recyclable 类别：
- **Recall** ≈ 1（几乎全部识别正确）
- **Precision** ≈ 0.9（一些 other 被误判为 recyclable）
- **F1 ≈ 0.95

**宏平均 F1 = (0.95 + 0) / 2 = 0.48** ✅ 符合日志！

---

## 📈 这说明了什么？

### ✅ 好消息
- **没有数据泄露**
- train/val 划分正确
- 模型正常学习

### ⚠️ 坏消息
- **严重的类别不平衡**
- 模型学会偏向预测多数类
- **对少数类（other）效果很差**

### 这在真实场景中很常见
- 垃圾识别中，可回收物通常远多于其他垃圾
- 这是**数据集问题，不是代码问题**

---

## 🎯 解决方案

### 方案 1：数据层面（推荐）
1. **收集更多 other 类别的数据**
   - 目标：至少 500-1000 张
   - 使比例接近 1:1 或 2:1

2. **使用数据重采样**
   - 对 recyclable 进行下采样（减少到 500 张）
   - 或对 other 进行上采样（数据增强）

### 方案 2：算法层面
1. **类别加权损失**
   ```python
   weight = [1.0, 1.0, 1.0, 15.0]  # 给 other 类 15倍权重
   criterion = nn.CrossEntropyLoss(weight=weight)
   ```

2. **修改评估指标**
   - 使用 F1-score 而非 Accuracy
   - 关注混淆矩阵

### 方案 3：演示层面
1. **诚实报告**
   - 说明数据不平衡问题
   - 展示真实性能（F1-score）

2. **重新训练平衡数据集**
   - 每类使用 500 张图像
   - 重新训练模型
